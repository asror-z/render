 What is LoadGo?

LoadGo is a Javascript plugin that allows you to create a progress bar using your own images.

- Perfect for logo animation when user is waiting for something to be loaded (a website, retrieving information, updating status, etc.)

- It creates an overlay above your img element and simulates a loading process using dimension changes.

- Tested up in IE 9, IE 10, IE Edge, Google Chrome and Mozilla Firefox.

 How to use LoadGo

If you are using jQuery, you have to include it BEFORE using this plugin. It works with jQuery >= 1.11.2.

If you are not using jQuery, just include LoadGo script and start using it.

 Production environment (CDN)

 Development environment

1. Download LoadGo from this link or clone it from GitHub.

2. Uncompress it (if zipped) and copy LoadGo folder into your JS scripts.

3. Insert the following code in your webpage: 

  1. <!-- If you use jQuery -->
  2. <script type="text/javascript" src="loadgo/loadgo.js"></script>

  4. <!-- If you don't use jQuery -->
  5. <script type="text/javascript" src="loadgo/loadgo-nojquery.js"></script>

You can also use the minified version: 

  1. <!-- If you use jQuery -->
  2. <script type="text/javascript" src="loadgo/loadgo.min.js"></script>

  4. <!-- If you don't use jQuery -->
  5. <script type="text/javascript" src="loadgo/loadgo-nojquery.min.js"></script>

Important LoadGo needs fully loaded images in order to read its dimensions correctly. If you are not sure when this is going to happen, you can use this piece of code with each image: 

  1. //jQuery
  2. $("#my-image").load(function() {
  3.   $('#my-image').loadgo();
  4. }).each(function() {
  5.   if(this.complete) $(this).load();
  6. });
  1. // Javascript
  2. var image = document.getElementById("my-image");
  3. image.onload = function () {
  4.   Loadgo.init(this);
  5. };

 Documentation

 Introduction

LoadGo is a plugin which provides you a better way to keep your users updated about a loading process. Some use cases may be:

  • User uploads a file to your server.
  • System is converting a file to PDF.
  • Current page is loading.

Important This plugin does not control asynchronous behaviour for your loading process, so you have to do that by yourself in your app.

To do this, LoadGo wraps your image into a div, and add an overlay inside this wrapper. Then, it will change overlay's dimensions when setting a progress to simulate a loading behaviour.

LoadGo will work only on img elements given by id.

This piece of code is a minimum example:

  1. <div>
  2.   <img id="logo" src="logo.png" alt="Logo" />
  3. </div>
  1. // jQuery
  2. $('#logo').loadgo();

  4. // Javascript
  5. Loadgo.init(document.getElementById('logo'));

 Initialization

LoadGo needs to be initialized in an img element before use.

jQuery 

  1. $('#logo').loadgo();

Javascript 

  1. Loadgo.init(document.getElementById('logo'));

Now, you are capable of set progress and simulate any kind of progression. LoadGo have methods and a couple of options which will help you.

LoadGo also applies an empty CSS class to overlay called loadgo-overlay in case that you want to implement your own resize function or some other advanced features.

 Options

  • bgcolor: background overlay color in hexadecimal or RGB. Default is #FFFFFF.
  • opacity: overlay transparency. Default is 0.5
  • animated: true if setprogress CSS width transitions are enable, false otherwise. Default is true (remember that Internet Explorer does not support CSS transitions).
  • image: image url to be used if you want a background image instead of a simple color. This option disables bgcolor option.
  • class: CSS class which will be applied to overlay. Make sure that all looks good because some CSS options could invalidate other LoadGo plugin CSS options. Default is null.
  • resize: resize function. LoadGo provides a function which automatically resizes LoadGo overlay by default, but you can use your own.
  • direction: animation direction. Possible values: lr (left to right), rl (right to left), bt (bottom to top), tb (top to bottom). Default is lr.
  • filter: CSS image filter according to CSS filter property. Possible values: blur, grayscale, sepia, hue-rotate, invert, opacity.

 Methods

  • Get / Set options: Get/Set plugin options. When setting options, the parameter must be a JSON object. This is specially useful if element is already initiate, and you want to change properties dynamically based on your app's logic. 

  1. // jQuery - Get current options
  2. $('#logo').loadgo('options');

  4. // jQuery - Set options
  5. $('#logo').loadgo('options', { direction: 'bt' });
  1. // Javascript - Get current options
  2. Loadgo.options(document.getElementById('logo'));

  4. // Javascript - Set options
  5. Loadgo.options(document.getElementById('logo'), { direction: 'bt' });

  • Set Progress: Set progress number to loading overlay. This number must be between 0 and 100. 

  1. // jQuery
  2. $('#logo').loadgo('setprogress', 50);
  1. // Javascript
  2. Loadgo.setprogress(document.getElementById('logo'), 50);

  • Reset progress: Reset current progress. This is really useful when you are using the same element for multiple loads, and you need to reset all before starting a new one. 

  1. // jQuery
  2. $('#logo').loadgo('resetprogress');
  1. // Javascript
  2. Loadgo.resetprogress(document.getElementById('logo'));

  • Get progress: Get current progress. This number will be between 0 and 100. 

  1. // jQuery
  2. $('#logo').loadgo('getprogress');
  1. // Javascript
  2. Loadgo.getprogress(document.getElementById('logo'));

  • Loop: Sets overlay to loop indefinitely until stopped. This is useful for situations where you have no way of measuring the progress. This method accepts a duration (ms) parameter to customize animation speed. 

  1. // jQuery
  2. $('#logo').loadgo('loop', 10);
  1. // Javascript
  2. Loadgo.loop(document.getElementById('logo'), 10);

  • Stop: Stops the loop previously initiated with loop method and shows the full image. Since loops are indefinite we need to use this method to manually stop it. 

  1. // jQuery
  2. $('#logo').loadgo('stop');
  1. // Javascript
  2. Loadgo.stop(document.getElementById('logo'));

  • Destroy: Removes all plugin properties and restore everything. This method will also removes div elements created (loadgo-container and loadgo-overlay), so make sure that you are not using them for anything else before calling destroy. 

  1. // jQuery
  2. $('#logo').loadgo('destroy');
  1. // Javascript
  2. Loadgo.destroy(document.getElementById('logo'));

 Real example

In your webpage, you are using a jQuery plugin like Uploadify to give your users a way to upload files to you page (for example: update his/her web avatar). Most of these plugins provide events like onUploadStart, onUploadProgress or onUploadComplete. These events have variables which give you a lot of information about your current load progress (file size, current uploaded bytes, etc).

You can use this information with LoadGo to update logo overlay like this:

  1. // Set LoadGo on your Logo
  2. $('#logo').loadgo();

  4. // Set Uploadify on your upload input
  5. $('#uploadinput').uploadify({
  6.   // init options...
  7.   onUploadStart: function (event) {
  8.     // Upload is going to start, so we need to reset loadgo
  9.     $('#logo').loadgo('resetprogress');
  10.   },
  11.   onUploadProgress: function (event) {
  12.     // We receive some bytes on our upload and updates loadgo progress,
  13.     // but first, we should calculate total uploaded percentage
  14.     var p = event.bytesLoaded / event.bytesTotal;
  15.     $('#logo').loadgo('setprogress', p);
  16.   },
  17.   onUploadComplete: function (event) {
  18.     // Upload complete
  19.   }
  20. });

 Current LoadGo examples

To create LoadGo minimum examples, some intervals were set to recreate this effect. Following code is used by CocaCola example at the beginning of this page:

  1. $(document).ready(function () {
  2.   $('#cocacola').loadgo();
  3. });

  5. // We create an interval which will be removed when progress completes
  6. var cocacolaInterval;
  7. function mainDemo () {
  8.   var p = 0;
  9.   $('#cocacola').loadgo('resetprogress');   // reset progress to start again
  10.   cocacolaInterval = window.setInterval(function (){
  11.     if ($('#cocacola').loadgo('getprogress') == 100) {
  12.       // When progress completes, we have to clear the interval
  13.       window.clearInterval(cocacolaInterval);
  14.     }
  15.     else {
  16.       var prog = p*10;
  17.       $('#cocacola').loadgo('setprogress', prog);
  18.       p++;
  19.     }
  20.   }, 150);
  21. };

mainDemo function is actually bind to 'Start' button, so when you click on it this function will be executed.
